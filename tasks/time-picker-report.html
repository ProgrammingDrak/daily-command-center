Unified time & date picker

Daily Command Center · June 28, 2026 · PR #163

At a glance

Setting a time anywhere in the Command Center now uses one clean, consistent control: an iOS-style scroll wheel for times and a tap-friendly month calendar for dates. The old fiddly clock-face dial (two clicks just to set one time) is gone, replaced everywhere at once.

Quick start
  1. Click any time on a task card, in the done modal, the block editor, the brief, or the day-review actual-time editor.
  2. Spin the hour / minute / AM-PM columns (drag, mouse wheel, or click a value). The selected time sits in the highlighted center band and shows large at the top.
  3. Tap Now to jump to the current time, then Set to confirm.
  4. For dates (reschedule, schedule-a-task, catch-up), the same control opens a month calendar — click a day, or hit Today.

Example scenarios

Situation: You want to pin a task to start at 2:30 PM.

Action: Click the start time on the card, spin to 2 : 30 : PM, hit Set.

Outcome: One smooth interaction instead of the two-step clock dial.

Situation: Logging the actual time you spent in day-review.

Action: The start/end fields now open the same wheel instead of a raw browser time box.

Outcome: Identical look and feel to every other time control.

Situation: Rescheduling a task to a future date.

Action: The date field opens the month calendar; click the day or use Today.

Outcome: Consistent, themed calendar instead of the OS-default date input.

What to watch for
  • On mobile / narrow screens the picker docks as a bottom sheet instead of floating next to the field.
  • The wheel uses native momentum scrolling — buttery on touch and the mouse wheel. If you'd prefer click-to-step arrows for precise mousing, that's an easy add-on.
  • Minute granularity is 1 minute (any time of day is reachable).

Summary

Replaced three inconsistent time/date input mechanisms with one shared module used across the whole app. Time picking is an iOS-style scroll wheel; date picking is a month-grid calendar. Every native time/date input is auto-upgraded, so the change is consistent everywhere with no per-screen rework.
